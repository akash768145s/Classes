üß† How Virtual DOM is Working?

Listen carefully friends! Virtual DOM is like making photocopy of your document before writing on it!

  1. First it is keeping one copy of DOM in memory only (not in real page)
  2. When change is coming, it is making new copy with changes
  3. Then it is comparing old copy and new copy to see what exactly changed
  4. Finally it is updating ONLY the changed parts in real DOM
  5. This way, very less work is happening! Very efficient!

Simple example: If you are changing one letter in big book, you are not rewriting full book! You are only changing that one letter only! Same logic!
